## [3.1.0] — Unreleased

### Added
- **Auto-detect watermark** (T29 Phase 2): a canvas-toolbar "Auto-detect" toggle runs a pure-JS edge-detection + connected-components pipeline directly in the browser to locate text-shaped watermark regions automatically. Displays a translucent magenta mask + bounding boxes as a display-only overlay (`#detectOverlay`) — never mutates the image. Shows "Found N candidate regions" in the canvas hint. Honours active region selections. Detection uses per-channel Sobel (max across R/G/B — catches iso-luminant colour watermarks that luminance-only Sobel misses), morphological close, BFS connected-component labelling, and text-vs-solid classification via hole-fill ratio. Includes large-image guard and a 0–1 confidence score contract for the future confirm→fill step (T43 / #44).
- **X-ray view selection aid** (T29 Phase 1): a canvas-toolbar "X-ray" toggle reveals faint or semi-transparent watermarks by applying `invert(1) contrast(1.6)` to the display — display-only, never alters pixels (eyedropper, smart fill, and export read the true image). Useful for light-on-white text watermarks that are otherwise too low-contrast to eyedropper accurately.
- **Smart fill: BFS / Fast-Marching geodesic fill** (T42, fixes #32, guard for #31): replaces the cardinal distance-weighted scan with a multi-source BFS that follows connected (geodesic) paths from the mask boundary inward. Fixes corner/edge pixels left unfilled on full-perimeter watermarks (#32). Handles concave masks and textured/gradient backgrounds without seams. Adds `options.maxFillRatio` (engine) / `0.8` (GUI): when the fill set exceeds 80% of the image region the fill is skipped and the canvas-hint pill turns amber — prevents the main-thread glitch at extreme tolerance (#31 partial).
- **Smart-fill edge dilation** (T30): smart fill now expands the colour-match mask by 1px before reconstructing, so anti-aliased watermark/text edges are filled from the true background instead of surviving as a faint halo. Automatic when Smart fill is on — no new control. Engine: `smartFill` gains an `options.dilate` radius (default 0 preserves legacy behaviour; the GUI passes 1).
- **Custom colour-picker popover** (T28): the "+" button in Panel 2 now opens a bespoke colour picker anchored directly to the button. Includes a saturation/value square, hue slider, hex field (primary), and RGB triplet (secondary) — all simultaneously visible, no mode switching, no HSL inputs. Replaces the native `<input type=color>` which Chrome pinned to the viewport top-left. Colour is committed to recents only on confirm; Esc / outside-click cancels with no history change.
- **Cursor-as-loupe eyedropper** (T27): while the eyedropper is armed the system cursor is hidden and the pixel-zoom magnifier loupe is centred exactly on the pointer. The centre crosshair cell is the pixel that will be picked on click, eliminating the alignment gap of the previous 18px-offset design. Grid, adaptive grid lines, and crosshair rendering unchanged; Esc / re-click disarms and restores the cursor.
- **`detectWatermark` calibration levers** (#45): two new options for `detectWatermark` — `blurRadius` (0/1/2 = none/3×3/5×5 separable box pre-blur before Sobel, to suppress photo texture) and `minAspect` (minimum bbox width/height ratio; 0 = off). Both default to no-ops so all existing call-sites are unaffected. Accompanied by a reproducible calibration harness (`scripts/calibrate-detect.js`) that sweeps all lever axes across three real fixtures + one adversarial synthetic. GUI overlay detection profile unchanged (`{edgeThreshold:150, preContrast:false}`).

### Fixed
- **`detectWatermark` `preContrast` default was dead** (#45): `buildLut(!!opt.preContrast)` ignored `DETECT_DEFAULTS.preContrast`, so the documented default-ON behaviour never applied — both the GUI and all option-less callers ran raw Sobel. Now uses `opt.preContrast != null ? opt.preContrast : DETECT_DEFAULTS.preContrast`. GUI overlay passes `preContrast:false` explicitly to preserve its pre-fix behaviour.
